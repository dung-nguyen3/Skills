**USE SOURCE FILE ONLY for content:**
- Facts and definitions

**USE YOUR MEDICAL TRAINING (built-in knowledge) for:**

- **High-yield exam points** - Board-relevant information and clinical significance
- **"If X think Y" clinical associations** - Pattern recognition and clinical reasoning

## DOCUMENT STRUCTURE

### Required Tabs (Sections)
The Word document must have 4 main sections (tabs):

#### Learning Objectives
- Each learning objective gets its own subsection
- Format:
  ```
  Learning Objective #X: [verbatim text from source]

  Summary: Myelodysplastic syndrome is a malignant stem cell disorder causing bone marrow failure. It primarily affects patients over age 60 and is 80% idiopathic, with 20% related to chemotherapy or radiation. Key diagnostic criteria include cytopenias, 1-20% blasts in bone marrow, and clonal hematopoiesis.

  [TABLE 1: Category/Details table OR Comparison table]

  [TABLE 2: Additional tables as needed]

  [Clinical Pearls & High-Yield Points Box]

  [Memory Tricks/Mnemonics Box - GOOD mnemonics only!]

  [Analogy Box]
  ```

#### Key Comparisons
- Side-by-side comparison tables
- Focus on commonly confused or similar conditions
- Examples:
  - Testicular Torsion vs Epididymitis
  - Phimosis vs Paraphimosis
  - Epispadias vs Hypospadias
  - Indirect vs Direct vs Femoral Hernias

## Key Comparison Charts Guidelines
- Create additional key comparison charts for information with similar categories that need differentiation
- Look for patterns across topics that students need to compare and contrast
- Examples:
  - For drugs: Compare mechanisms of action (all drugs have different MOAs - help differentiate)
  - For clinical exams: Compare techniques (e.g., pelvic exam technique vs rectal exam technique)
  - For conditions: Compare diagnostic criteria, symptoms, treatments when similar
  - For Physiology: FEV vs FVC, vs TLC, etc.
- Create comparison charts for any category that appears across multiple topics/drugs/conditions
- Goal: Help students differentiate between similar concepts with side-by-side comparisons
- **REQUIRED for Drug Mechanism comparisons:** Add "Analogy" column/row with 2-4 sentence analogy explanation for EACH drug's mechanism to help students understand how it works (e.g., "Like cutting the assembly line belt"). Research good analogies using web search.


#### Master Chart
- Comprehensive table of ALL conditions/topics
- Columns: Condition | Key Features | Additional columns
- Organized by category or alphabetically
- Color-coded rows for visual organization or to group drug classes, etc.
- No page references needed

#### High-Yield Summary
- Color-coded boxes grouped by category:
  - Anatomy Must-Knows
  - Emergency/Critical Conditions
  - Normal Variants
  - Clinical Pearls
  - Pain Characteristics
  - Diagnostic Tests

## COMPARISON TABLE GUIDELINES

### When to Create Comparison Tables
Create comparison tables when there are **2 or more** similar:
- Conditions with overlapping presentations
- Drugs in the same class
- Categories or classifications
- Exam findings or techniques

### Common Comparison Categories
Choose relevant categories from:
- **Age/Epidemiology**: Age groups, prevalence, demographics
- **Risk Factors**: Predisposing factors, causes
- **Presentation**: Symptoms, signs, appearance
- **Location/Anatomy**: Anatomical positions, affected structures
- **Diagnosis**: Diagnostic criteria, tests, findings
- **Management**: Treatment approaches (only if in source)
- **Complications**: Outcomes, risks
- **Laboratory Values**: Test results, thresholds (only if in source)
- **Prognosis**: Expected outcomes
- **Key Distinguishing Features**: What sets them apart
- Other categories as needed to answer learning objective.

### Table Formatting
- **Headers**: Bold white text on colored background
- **Row labels** (first column): Bold text on light colored background
- **Data cells**: Regular text on white background
- Font: CALIBRI, size 12pt (11pt for dense tables)
- Table style: 'Table Grid'

## PYTHON-DOCX IMPLEMENTATION

### Required Functions

```python
from docx import Document
from docx.shared import Pt, RGBColor, Inches
from docx.enum.text import WD_ALIGN_PARAGRAPH
from docx.oxml.ns import qn
from docx.oxml import OxmlElement

def set_cell_background(cell, color):
    """Set cell background color using hex color string"""
    shading_elm = OxmlElement('w:shd')
    shading_elm.set(qn('w:fill'), color)
    cell._element.get_or_add_tcPr().append(shading_elm)

def set_cell_text(cell, text, bold=False, color=None, size=11):
    """Set cell text with formatting"""
    cell.text = text
    for paragraph in cell.paragraphs:
        for run in paragraph.runs:
            run.font.size = Pt(size)
            run.font.name = 'Calibri'
            run.bold = bold
            if color:
                run.font.color.rgb = RGBColor(*color)

def add_colored_box(doc, title, content_list, title_color, bg_color='F3E5F5'):
    """Add a colored information box"""
    para = doc.add_paragraph()
    para.paragraph_format.left_indent = Inches(0.3)
    para.paragraph_format.space_before = Pt(6)
    para.paragraph_format.space_after = Pt(6)

    for item in content_list:
        para.add_run(f'{item}\n').font.size = Pt(11)

    return para
```

## TROUBLESHOOTING

### Python-docx Common Errors

**Error: `'Document' object has no attribute 'runs'`**
```python
# Wrong:
doc.add_heading('Title', 1)
doc.runs[-1].font.color.rgb = RGBColor(118, 75, 162)

# Correct:
heading = doc.add_heading('Title', 1)
heading.runs[0].font.color.rgb = RGBColor(118, 75, 162)
```

**Error: Table alignment issues**
- Always use `table.style = 'Table Grid'`
- Set cell text AFTER setting background color
- Use `set_cell_background()` before `set_cell_text()`
